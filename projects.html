Project - Coding in C

During my time in the Harvard CS50 course, I tackled a variety of tasks, including those involving programming challenges. As everybody I started from a simple program that displays the message "Hello, World!" on the screen.

Then I write a few programs:

  • Cash: A program that calculates the minimum number of coins needed to make change for a specified amount of money, utilizing different coin denominations.
  • Mario (pyramid): A program that draws a pyramid with dots and hashes based on the height provided by the user.
  • Caesar: A program that encrypts text using the Caesar cipher, shifting letters by a certain number of places in the alphabet.
  • Substitution: A program implementing a substitution cipher algorithm, where each letter in the text is replaced with another letter according to a given key.
  • Readability: A program that calculates the average number of sentences per page and the average number of letters per 100 words in a text to assess its readability.
  • Speller: A program that checks the spelling of text by comparing it with a loaded dictionary.
  • DNA: A program that analyzes DNA sequences and determines which DNA sequences match a given pattern, using text analysis algorithms.
  • Foto converter: programs that involved image manipulation. These programs allowed me to perform various operations on images, such as flipping them, blurring them, converting them to grayscale or sepia tones, and applying other filters and effects. For example, one of the programs I worked on might take an input image and apply a blur effect to create a softened appearance. Another program might convert a color image to grayscale, reducing it to shades of black and white. These exercises helped me understand concepts related to image processing and algorithms for manipulating visual data.
  • Credit: which focused on credit card validation. In this task, the goal was to create a program capable of verifying whether a given credit card number is valid according to the Luhn algorithm. This involved implementing algorithms to check for the card's validity based on its structure and specific mathematical criteria. It was an engaging exercise that deepened my understanding of algorithms and data validation techniques.

These tasks allow for the development of programming skills, understanding of algorithms and data structures, and practical application of these skills in the context of computer science problems. You can find my solutions to these tasks on my GitHub profile.

Project - Finance App

C$50 Finance is my first web application designed to facilitate the management of stock portfolios. With this tool, users can monitor real-time stock prices and track the value of their portfolios. Additionally, users have the ability to buy and sell stocks directly through the application by querying for stock prices.

The application leverages external APIs, such as IEX, to retrieve stock quotes. For example, by using a URL like https://api.iex.cloud/v1/data/core/quote/nflx?token=API_KEY, users can obtain data for a specific stock, in this case, Netflix (symbol: NFLX). The API returns the requested data in JSON format, which the application processes to display relevant information to the user.

Overall, C$50 Finance provides users with a convenient and efficient way to manage their stock portfolios, offering real-time data and trading capabilities in a user-friendly interface.
